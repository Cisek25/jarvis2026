Client deliverables for IdoSell panels (body_top, body_bottom, CSS, SEO_TYTULY.md) must be clean copy-paste content. No emojis, no decorative header bars (===/---/~~~), no comments referencing panel URLs or CMS field names, no character counters, no "generated by JARVIS" signatures, no box-drawing Unicode separators.

**Why:** EcoCamping 2026-04-22 — GLOWNA_PL__body_top.html would not save in the IdoSell panel. User manually stripped my emoji and a comment like "Panel: client57820.idosell.com → Strona główna CMS" and the save immediately worked. IdoSell's HTML save endpoint has a WAF/sanitizer that rejects non-ASCII symbolic characters and strings resembling injection/instructions in the content field.

**How to apply:** When generating any file that will be pasted into an IdoSell panel:
- HTML comments only with plain section names: `<!-- Hero -->`, `<!-- Oferty -->`
- No URLs, panel IDs, dates, or tool attribution in comments
- SEO files: section header + 4 fields with colons — nothing else (see JARVIS instinct 009)
- Before delivery, grep must return 0 for both: emoji ranges (U+1F300–U+1FAFF, U+2600–U+27BF) and patterns `client\d+\.idosell\.com|Panel:|generated by`
- Applies to all JARVIS-generated client work without exception — codified in JARVIS instinct 027
