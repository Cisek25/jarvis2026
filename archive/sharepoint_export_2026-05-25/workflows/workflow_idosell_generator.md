## Workflow
1. Client fills brief (DOCX/HTML)
2. Claude creates config.json from brief
3. `python generate.py config.json` → 5 files in DO_WKLEJENIA/
4. Claude customizes homepage content, subpages
5. User pastes into panel per INSTRUKCJA.txt
6. Claude tests live via Chrome DevTools
7. Post-mortem → trap tracker update

## WCAG features (added 2026-04-10)
- Skip-to-content link (§25 CSS + §15 JS)
- .sr-only class
- focus-visible on all interactive elements
- prefers-reduced-motion (kills ALL animations)
- prefers-contrast:high
- Touch targets min 44px (@media pointer:coarse)
- FAQ: role=button, tabindex, aria-expanded, aria-controls, keyboard Enter/Space
- Blog cards: focus-visible outline

## Blog system (added 2026-04-10)
- CSS: §26 — grid, cards, date/title/excerpt/cta, responsive
- JS: §16 initBlog() — auto-fetches .news-item from system, builds cards
- Pattern: WCA auto-listing (fetch homepage → parse .news-item → build cards)
- HTML: minimal container div needed, JS populates from panel blog posts
- System blog: Panel → Blog/Aktualnosci → Dodaj nowy wpis
