### CSS specificity hack:
Custom.css klienta używa `.nav-link.nav-link` (chained class) + `:not()` chains z wysoką specificity. Aby pobić używamy **chained class names** (`.foo.foo`) lub matchującego selektora 1:1.

Przykład: `html body footer .footer__contact.footer__contact` (2× `.footer__contact` = +1 specificity boost).

### Reguła `font-size:13px` dla menu (§103b):
Selector użyty: `html body header.default13 .nav-link.nav-link, html body header.default13 .navbar a[href]:not(.logo):not(.navbar-brand):not([class*="btn"]):not(.language__toggler)` — kopia z custom.css aby cascade order zdecydował.

### Lessons learned:
- `--fr-dark` musi być w `:root` — wcześniej zapomniany, sekcje dark renderowały się cream
- `<header>` w `.fr-compare-model` był łapany przez systemowy `header { position: fixed }` — fix przez override `position: relative` na `html body article header`
- System Idosell hides `.footer__social` + `.powered_by` przez `display: none !important` — fix przez chained class boost
