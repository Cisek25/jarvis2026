- **Trap #26 / instinct 015 + 026 MASTER** — featured offers Pattern A (parse `.container-hotspot`, retry [0,400,1200,2500] + 4s guard, ZERO hardcoded fallback, hide section gdy 0)

**False positives walidatora (10 UX critical)** — udokumentowane:
- **9× UX-020** "Biały tekst na jasnym tle" — walidator nie wykonuje CSS cascade analysis. Wszystkie wystąpienia to `color: #FFFFFF` na elementach wewnątrz `.fr-final-cta { background: dark }` lub `.fr-page-hero { background-image: ...; ::after { dark overlay } }`. **Faktyczny kontrast** w produkcji: 19.6:1 (AAA). Live audit (FAZA 7) potwierdzi.
- **1× UX-002** "img z pustym alt=''" — to JS template w `FR_KONIEC_BODY.html` (`<img loading="lazy" src="' + escapeAttr(o.imgSrc) + '"`...). Runtime alt jest podstawiany z `o.title.replace(/"/g, '&quot;')`. False positive.

## Known limitations / ryzyka

- **Zdjęcia atrakcji** to placeholdery Unsplash (nie są autentycznymi zdjęciami Wrocławia). Atrakcje wyglądają poprawnie, ale nie zawsze pokazują wskazaną w tytule lokalizację. **Trzeba podmienić** zgodnie z `ZDJECIA_LINKI.md` po pobraniu z Pexels lub Wikimedia (uwaga: hotlinking Wiki = 429, więc tylko **pobranie do galerii**).
- **Featured offers** pokazują się gdy klient zaznaczy w panelu. Jeśli 0 wyróżnionych → sekcja `display: none` (instinct 015).
- **PriceLabs (z briefu)** — to backend integration, nie dotyczy frontu strony. Wzmianka w `OBSLUGA_NAJMU_PL/EN` w sekcji "Marketing i widoczność" jako USP.
- **Pakiet wdrożeniowy z briefu** nie był skonkretyzowany, dostarczono pełen scope (8 stron × 2 języki). Zakres można skracać/rozszerzać.
- **Polityki, regulamin, polityka prywatności** — system IdoBooking obsługuje automatycznie, widoczne w stopce.
- **Adres kontaktowy w stopce** to firmowy `Białowieska 69/11`. Pochodzi z systemu (wgrane w panelu przez klienta), nie wpisujemy ręcznie w plikach.
