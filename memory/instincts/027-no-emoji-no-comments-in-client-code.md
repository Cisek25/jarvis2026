# INSTYNKT 027: ZERO emotikonek i ZERO komentarzy JARVIS w kodzie klienta

## Reguła
Pliki wklejane do panelu IdoSell (body_top, body_bottom, CSS, SEO_TYTULY) NIE MOGA zawierac:
- Emotikonek (UTF-8 emoji: ktory kolwiek symbol z zakresow U+1F300-U+1FAFF, U+2600-U+27BF, U+2300-U+23FF)
- Naglowkow ozdobnych z ciagami znakow typu `===`, `---`, `~~~`, `###`
- Komentarzy typu "Panel: clientXXXXX.idosell.com -> Strona glowna CMS"
- Komentarzy typu "wklej to w pole body_top", "skopiuj i wstaw", "instrukcja wklejania"
- Liczenia znakow w nawiasach ("META Tytul: ... (58 znakow)")
- Uzasadnien SEO, adnotacji "generated by JARVIS", podpisow autora
- Seperatorow sekcji z Unicode box-drawing (U+2500-U+257F)

## Dlaczego
**IdoSell WAF/parser HTML odrzuca zapis pola "Tresc" gdy wykryje podejrzane znaki lub zawartosc.**
Klient EcoCamping (2026-04-22): GLOWNA_PL__body_top.html z emotikonka + komentarzem
"Panel: client57820.idosell.com -> Strona glowna CMS" NIE ZAPISYWAL SIE. Po usunieciu obu
elementow recznie przez klienta — zapis zadzialal.

Prawdopodobnie WAF IdoSell blokuje:
- Non-ASCII symbole w pierwszych bajtach pola
- Stringi przypominajace instrukcje injection/odbiorcze ("Panel:", "CMS", URL-e w komentarzach)

## Jak stosowac
Przy generowaniu KAZDEGO pliku HTML/CSS dla klienta:
1. `<!-- -->` komentarze tylko z nazwa sekcji: `<!-- Hero -->`, `<!-- Oferty -->`. Bez URL, bez nazwy panelu, bez emoji.
2. CSS komentarze `/* */` tylko z nazwa sekcji: `/* Layer 3: theme */`. Bez daty, bez autora.
3. NIGDY symboli `~`, `=====`, ozdobnych ramek
4. SEO pliki (SEO_TYTULY_*.md) — tylko naglowek sekcji + 4 pola z dwukropkiem (patrz instinct 009)
5. Przed zapisem pliku: grep regex na U+1F000-U+1FFFF zwraca 0 matchy

## Weryfikacja
```bash
# Sprawdz czy plik ma emotikonki
LC_ALL=C grep -P "[\x{1F300}-\x{1FAFF}\x{2600}-\x{27BF}]" plik.html

# Sprawdz komentarze z URL panelu
grep -E "client[0-9]+\.idosell\.com|Panel:|CMS" plik.html

# Oba powyzsze MUSZA zwrocic 0 wynikow przed oddaniem klientowi
```

## Wyjatki
Brak. Nawet `<!-- JARVIS v1.14 -->` jest zabronione. Czysty kod, bez sladow narzedzia.
